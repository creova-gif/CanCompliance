# 1. DATA MAPPING (CANADIAN REQUIREMENT)

Map ALL personal and business data:

### Categories:

* User data (name, email, login info)
* Client data (contacts, companies)
* Proposal data (documents, pricing)
* Behavioral data (tracking, analytics)

For EACH:

* Where is it stored?
* Is it inside or outside Canada?
* Who has access?

FLAG:

* Cross-border data transfers (CRITICAL under PIPEDA & Law 25)

---

# 2. CONSENT & PRIVACY (PIPEDA CORE)

Verify:

* Explicit user consent before collecting data
* Clear purpose of data usage
* Ability to:

  * Access data
  * Correct data
  * Delete data

Test:

* Is consent granular or forced?
* Are users informed clearly?

FAIL if:

* Data is collected without clear explanation

---

# 3. EMAIL & COMMUNICATION (CASL)

Audit ALL messaging:

* Proposal emails
* Notifications
* Marketing emails

Verify:

* Explicit opt-in consent
* Clear unsubscribe mechanism
* Sender identification

FAIL if:

* Any email can be sent without consent

---

# 4. QUEBEC LAW 25 (HIGH STANDARD)

If ANY Quebec users:

Verify:

* Data minimization (only collect what is needed)
* Privacy by default
* Data protection impact assessment (DPIA)
* Ability to delete personal data permanently

FLAG:

* Missing privacy controls

# 5. AUTHENTICATION & ACCESS CONTROL

* Secure login (JWT/OAuth)

* Role-based access (admin, user, team)
